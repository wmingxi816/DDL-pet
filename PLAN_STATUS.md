# DDL鼠鼠 Plan Status

更新时间：2026-05-18  
当前分支：`main`  
最近功能提交：本文档所在提交

## 当前目标

构建一个本地优先的 Android MVP：任务打卡、DDL 提醒、宠物养成三部分先跑通主流程。首版不接账号、云同步、AI 文案、真实定位和真实 PK。

## 已完成

- 原生 Android 项目已初始化，包名为 `com.ddlmouse.app`，应用名为 `DDL鼠鼠`。
- 技术栈已接入 Kotlin、Jetpack Compose、MVVM、Room、DataStore、Navigation Compose。
- 底部主导航已完成：`任务`、`宠物`、`我的`。
- 任务模块已定义：`每日任务`、`每周任务`、`每月任务`、`大项目`、`待办事件`。
- 难度与积分规则已实现：简单 `+5`、中等 `+12`、困难 `+25`、地狱 `+50`，逾期扣对应分值的 50%，积分最低为 0。
- 任务页已从单任务卡片改为栏目式结构：顶部概览、最近 DDL、五个大栏目、栏目内小任务行。
- 新增任务弹窗已按任务类型动态显示字段：
  - 每日任务：时间段、每日提醒时间。
  - 每周任务：每周出现日期、本期 DDL、提醒预览。
  - 每月任务：每月出现日期、本期 DDL、提醒预览。
  - 大项目：DDL、项目阶段、提醒预览。
  - 待办事件：可选 DDL、提醒预览。
- 新增任务字段已贯通到 ViewModel、Repository 和 Room：
  - 备注、循环模式、提醒开关、偏好提醒时间、时间段、每周日期、每月日期、项目阶段。
- 任务编辑能力已完成：
  - 任务行提供编辑入口。
  - 编辑弹窗复用动态任务表单，并预填已有标题、DDL、难度、备注、提醒和循环配置。
  - 保存后更新任务模板、同步当前未完成任务行，并重建提醒计划。
- Room 数据库已加入 `1 -> 2` 迁移，兼容新增任务字段。
- DDL 智能提醒策略已实现基础版本，并通过 `AlarmManager` 调度本地通知。
- Android 13+ 通知权限请求、通知渠道、提醒接收器已接入。
- 宠物页已完成 MVP：
  - 占位鼠鼠形象、积分、等级、心情、饱食度、气泡文案、喂食、互动、商店兑换。
- 我的页已完成 MVP：
  - 当前统计、提醒开关、附近 PK 占位入口。
- 每日 05:00 日界线、打开 App 初始化任务、结算昨日任务、每日总结弹窗已实现基础版本。
- 宠物多场景内置文案库已接入，不接 AI，保留后续替换空间。

## 部分完成

- 每周、每月任务的配置字段已经保存，但周期实例生成还没有完全按 `weeklyDays`、`monthlyDay` 做精细生成。
- 每日任务的提醒时间已经保存，但无 DDL 的每日固定提醒还没有完整调度。
- DDL 修改后可以通过任务编辑触发提醒重建，但还没有单独的多提醒编辑列表。
- 商店和宠物装扮已有数据模型和兑换流程，但装扮视觉表现仍是占位。
- PK 功能目前只有入口，没有定位、匹配、账号和对战结算。

## 未完成

- 账号系统、云同步、多设备同步。
- 真实鼠鼠图片素材或表情包资源接入。
- 宠物动画。
- AI 文案生成。
- 真实附近 PK。
- 任务搜索、筛选、归档和历史记录页。
- 完整的提醒时间手动编辑列表。
- 真机/模拟器 UI 自动化验证。
